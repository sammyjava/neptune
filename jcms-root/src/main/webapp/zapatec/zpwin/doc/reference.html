Chapter 1. General Information

The Zapatec DHTML Window is an easy, attractive, and versatile way to display information. It is also a reliable tool for handling interaction with users and present data in "windows-like" style. Our window works in different browsers, it consumes system resources very carefully and it takes very little coding to set it up on your page.

1. Full-Featured Windows

The Zapatec DHTML Window is available under commercial and lite licenses. If you use the lite license you need to include a link to http://www.zapatec.com on every single page that uses it.

The Zapatec DHTML Windows supports the standard window behaviour including:

  • Close

  • Minimize

  • Restore

  • Resize

  • Window Title Bar

  • Window Status Bar

  • Minimized windows on the bottom of the screen (web-page)

  • Different look for a top window

2. CSS-Based Themes

The Windows look is controlled via CSS. Choose from existing themes, or create your own. No programmer skills are needed to create your own theme. Select the one that suits your needs.

3. Rich API

Using the windows API you can control the following:

  • Size

  • Position

  • Minimum width and height

  • Content, which can include various HTML markup

  • Title

  • Status bar

After the window is created you can:

  • Hide it

  • Close it

  • Bring it to the front

  • Move it

  • Resize it

  • Change title and status messages

4. Types of Content Sources

Our windows correctly display HTML content, so you can use all possible <HTML> tags to create rich mark-up. Also our window accepts plain text as a source and adds horizontal and vertical scrollbars if text doesn't fit.

5. Multiple Browsers Support

The DHTML Windows code has been tested on Internet Explorer 6.0+, Opera 9.0+, Firefox 1.0+, and Safari 2.0+

6. Dynamic Fetching of Content

Window can fetch its content from server, so you don't have to include it inside your page.

7. Windows with Predefined Behavior

You can Use Windows with Predefined Behavior: alert box, confirmation window, generic dialog.

You can use them to notify visitor about some restrictions, make them confirm to something or simply give them a list of options to select from. See "alerts" demo.

Chapter 2.  Installation Procedure

To ensure proper functioning of this widget you should carefully follow the installation procedures.

1. Download DHTML Zapatec Window

The DHTML Windows files are bundled in a zip file. Click the download link in the Windows section of Zapatec web site, and follow the instructions to download the file.

Save the zpwin.zip file in your websites root folder on your computer or server.

2. Unpack zpwin.zip

  1. On your computer, go to the folder where you saved the zpwin.zip file.

  2. zpwin.zip contains a folder called zapatec where all of the files you will need to create Zapatec windows reside. Additional files that are required for windows proper functioning are hold in the utils folder.

  3. Open or double-click zpwin.zip to extract it.

Chapter 3. Quick Startup

Make sure you have downloaded and unpacked Zapatec Windows files. If you haven't done this yet, refer to the Installation Procedures section for instructions.

1. Insert the Headers

In your web editor (Dreamweaver, UltraEdit, etc.), open the web page where you want the windows placed. Insert your cursor before the ending </head> tag and paste the following script paths:

<!-- Javascript Zapatec utilities file -->  
<script type="text/javascript" src="utils/zapatec.js"></script>
<!-- Javascript file to support window -->
<script type="text/javascript" src="zpwin/src/window.js"></script>

1.1. Path Warning

If the web page containing the menu is saved in the zpwin folder, these lines will work without any changes. If, however, you saved your web page in another location (for example, directly under your website root folder), you need to adjust the path for these files.

Do not intend to change the logical structure of the zpwin.zip file.

2. Upgrade Warning

If you are migrating from 2.0 to the latest version, please be aware that the utils folder is removed from the zpwin folder. Now, it is located at the root directory, right where zpwin folder is.

Adjust paths to follow a new structure of our widget distribution package.

Chapter 4. Create the Window

The easiest way to create windows is to place the following code inside your <head> section:

<head>
    ...
    <script type='text/javascript'>   
        var win1 = Zapatec.Window.setup({
            left:50, 
            top:130, 
            raiseOnlyOnTitle:false, 
            content : '<span>Hi, I'm a Window content</span>'
        });
    ...
</head>

This window will show up when the page is loaded.

Sometimes you may want to display it on some other event. For example, on button click. You can trigger it manually or use popup option (described further).

As you see each configuration property (on this example: left, top, raiseOnlyOnTitle, etc.) has its definition listed after «:» and is separated from the next property definition with «,». The last property on the list shouldn't have «,» at the end.

1.  Configuration Options

1.1. theme (string)

Default value: 'default'

Specify the name of the CSS file that will be used as theme for your window. The predefined themes reside in the themes folder of your distribution package. If you are planning to apply a theme that does not belong to the themes folder, you should specify the full path to it.

1.2. showMinButton (boolean, default: true)

This option defines whether the Minimize button should be displayed for the current window. If set to FALSE, the Minimize button will not appear for this window.

1.3. showMaxButton (boolean, default: true)

This option defines whether the Maximize button should be displayed for the current window. If set to FALSE, the Maximize button will not appear for this window.

1.4. showCloseButton (boolean, default: true)

This option determines whether to display the Close button for the current window. If set to FALSE, the Close button will not appear for this window.

1.5. showStatus (boolean, default: true)

Use this option to define whether to show status bar text. If FALSE, the status bar won't be displayed for this window.

1.6. showTitle (boolean, default: true)

By using this option you may determine whether to show the title bar text of the window.

1.7. canResize (boolean, default: true)

If you do not want your window to be resizable, you should set the canResize option to FALSE. Otherwise, by providing TRUE in the constructor, you will enable your users to resize the window at the runtime.

1.8. raiseOnlyOnTitle (boolean, default: false)

Setting this option to FALSE allows your users to make the window active by clicking anywhere on it. Otherwise, the window will only gain focus when its title is clicked. By default it is set to FALSE.

1.9. canDrag (boolean, default: true)

Set this option to FALSE to disable the drag of the window.

1.10. dragMethod (string, default: cut)

Use this option to specify the method of dragging the Window. Provide "cut" to drag the Window itself, or "dummy" to drag the copy of your Window marked with a dashed border.

1.11. addDragCSS (string, default: "")

This option specifies the class name that will be added to the Window container while it is dragged.

1.12. newDragCSS (string, default: "")

The newDragCSS option specifies the class name that will overwrite all class names of the Window container while it is dragged.

1.13. modal (string, default: false)

If set to TRUE, the current window retains focus until you close it. Background will be shadowed when this modal window is open.

1.14. eventListeners

Use this option to specify an array of event listeners and functions to be called for the relevant events.

For example:

eventListeners : 
{
    onClose : someCloseFunction,
    onShow : someShowFunction,
    ...
}

Important

Do not put comma ( ",") after the last event listener.

For more information about available events, see the Events section.

1.15. onClose (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

This option specifies the custom event handler that will be called when the window is already closed.

1.16. onRestore (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

Use this option to specify your custom event handler that will be called when the Restore button is clicked.

1.17. onMinimize (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

By using this option you may define the custom event handler that will be called when the Minimize button is clicked.

1.18. onMaximize (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

This option determines what custom event handler will be called when the Maximize button is clicked.

1.19. onShow (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

The option allows you to specify your custom event handler that will be initiated when the show() method is called.

1.20. onHide (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

You can use this option set the custom event handler that will be initiated when the hide() method is called.

1.21. onResize (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

This option is used to specify your custom event handler that will be called when the window is resized.

1.22. onRaise (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

Use this option if you want to set the custom event handler that will be called when the window becomes active.

1.23. onContentLoad (function)

Deprecated. Use eventListeners option instead. Preserved for backward compatibility.

This option gives you the ability to specify your custom event handler that will be called when the content for the window is loaded.

1.24. limit (object)

This option limits the size and position of the window instance. The following object is expected:

limit : {
    minWidth : 100, // some integer value
    maxWidth : 500, //some integer value
    minHeight : 100, // some integer value
    maxHeight : 500, //some integer value
    minX : 100, // some integer value
    maxX : 500, //some integer value
    minY : 100, // some integer value
    maxY : 500 //some integer value
}

Important

Do not put comma (",") after the last value.

1.25. minWidth

Deprecated. Use limit option instead. Preserved for backward compatibility.

Use this option to set the minimal width of the window.

1.26. minHeight

Deprecated. Use limit option instead. Preserved for backward compatibility.

This option specifies the minimal height of the window.

1.27. iframeContent (boolean, default: false)

If set to TRUE, iframe will be specified as a type of container for the window content.

1.28. hideOnClose (boolean, default: false)

Use this option to define whether the window should be hidden or destroyed on close. If TRUE, the window will hidden. Otherwise, destroyed.

1.29. dragMin (boolean, default: false)

By setting this option you may determine whether the minimized window should be draggable. If set to TRUE, the minimized window can be dragged.

1.30. bottomMinimize (boolean, default: true)

The bottomMinimize option defines whether the minimized window should maintain its position or be collapsed to the bottom. If set to FALSE, the minimized window will remain fixed.

1.31. lang (string, default: "eng")

Use this option to set the localization language that will be applied to messages.

1.32. popup (string)

If defined, the window becomes popup window. Using this option you can pass an HTML element ID that will trigger popup window.

1.33. triggerEvent (string, default: click)

Use the triggerEvent option to define what event fired by trigger will be used to call popup window.

1.34. align (string, default: Bl)

You can align trigger element relatively to the trigger object. For the purpose of specifying certain alignment use two characters. The first character stands for vertical alignment; the second — for horizontal.

The following list outlines the letters used for alignment definition:

The first character in the align string may be one of the following:

  • T — positions the window completely above the reference element; window bottom margin aligns with the top margin of the element (bottom to top)

  • t — the window is positioned above the reference element but might overlap it; window bottom margin aligns with the bottom margin of the element (bottom to bottom)

  • C — the window appears vertically centered to the reference element and may overlap it depending on the horizontal alignment

  • b — the window is below the reference element but might overlap it; window top margin aligns with the top margin of the element (top to top)

  • B — positions the window completely below the element; window top margin aligns with the bottom margin of the element (top to bottom)

The second character in the align string is expected to be one of the following:

  • L — positions the window completely to the left of the reference element; window right margin aligns with the left margin of the element (right to left)

  • l — positions the window to the left of the element but may overlap it; window left margin aligns with the left margin of the element (left to left)

  • C — the window appears horizontally centered to the reference element but may overlap it depending on the vertical alignment

  • r — positions the window to the right of the element but may overlap it; window right margin aligns with the right margin of the element (right to right)

  • R — positions the window completely to the right of the element; window left margin aligns with the right margin of the element (left to right)

1.35. width (integer)

This option sets initial width of the window in pixels.

1.36. height (integer)

The height option defines initial height of the window in pixels.

1.37. left (integer)

Sets initial X coordinate of the window.

1.38. top (integer)

Sets initial Y coordinate of the window.

1.39. title (string)

Use this option to specify Window title.

1.40. content (string)

You can specify text content by passing string to this property.

1.41. divContent (string of object)

This option specifies an ID of the element that would be used as a Window content.

The original element is cut from the DOM tree and put inside Window.

See the Advanced Usage section for instructions on how to preserve original DOM element with the window content in its original place.

1.42. urlContent (string)

By using this option you may define the URL to load content from. It should be in valid HTML format.

When iframeContent is set to TRUE, then you can load any valid HTML page inside the window.

If iframeContent is FALSE, your source page should not contain tags that are not allowed inside the <BODY> section. See the Advanced Usage section for details.

1.43. initialState (string, default: "visible")

If set to "hidden", the created window will not be displayed until show() method is called.

1.44. resizeDirection (string, default: "all")

This option specifies the directions in which you can resize window.

Possible values: "all", "vertical", "horizontal", "top", "left", "bottom", "right", or any combination of them separated by "-". For example:

resizeDirection : "vertical-left"

1.45. fixed (boolean, default: false)

If set to TRUE, window remains in the same position on the screen while scrolling the page.

The behaviour is similar to setting CSS "position" attribute to "fixed" except it works in IE too.

1.46. template (string)

Default value: path to the struc.html file located in src folder

Configure this option to refer to the file that contains the HTML layout for the window.

See the Advanced Usage section for details.

1.47. buttonTheme (string)

This option controls look and feel of the buttons in the alert/confirm windows using a custom theme. The default value is 'default'.

1.48. message (string)

Use this option to provide the text that will be displayed in the alert/confirm window when it pops up.

2. Methods

2.1. getContainer()

This function returns window container.

2.2. setState(state)

This function sets the state of the window.

Parameters:

state {object} with the following properties:

  • x - left coordinate;

  • y - top coordinate;

  • width - width of container;

  • height - height of container;

  • url - URL of the content;

  • div - an id of the DIV element containing the content;

  • text - a text that should be used as a content.

2.3. getState()

Call this function to get the state of the window. Returned object has the following properties:

  • x - left coordinate;

  • y - top coordinate;

  • width - width of container;

  • height - height of container;

  • url - if content is set from URL, then it contains that URL, otherwise null;

  • div - if content is set from DIV element, then it contains its id, otherwise null;

  • text - if content is set as a text string, then it contains that text, otherwise null.

2.4. setPos(left, top)

Deprecated. Use setScreenPosition() instead. Preserved for backward compatibility.

This function sets the position of the window.

Parameters:

  • left {int} - x coordinate for the window;

  • top {int} - y coordinate for the window.

2.5. setPosition(left, top)

Use this method to set the window position relatively to the top left corner of the page.

Parameters:

  • left {int} - x coordinate for the window;

  • top {int} - y coordinate for the window.

2.6. setScreenPosition(left,top)

Specifies the position of the window on the browser screen.

Parameters:

  • left {int} - x coordinate for the window;

  • top {int} - y coordinate for the window.

2.7. getPosition()

Gets the position of the window. Returned object has the following properties:

  • x {int} - x coordinate for the window;

  • y {int} - y coordinate for the window.

2.8. getScreenPosition()

Call this function to get the window position on the browser screen. Returned object has the following properties:

  • x {int} - x coordinate for the window;

  • y {int} - y coordinate for the window.

2.9. moveFor(offsetX,offsetY)

By calling this function you can move your window for the specified offset.

Parameters:

  • offsetX {int} - x offset to move for;

  • offsetY {int} - y offset to move for

2.10. setWidth(width)

Use this function to set the width for the window.

Parameter:

width {int} - width of the window

2.11. setHeight(height)

This method sets the height for the window.

Parameter:

height {int} - height of the window.

2.12. getWidth()

Call this method to get the width of the window.

2.13. getHeight()

This function is invoked to get the height of the window.

2.14. setSize(width,height)

Use this function to set the size of your window.

Receives the following parameters:

  • width {int} - width for the window;

  • height {int} - height for the window.

2.15. setPosAndSize(left, top,width,height)

This function is called to set the size and position for your window.

Parameters:

  • left {int} - x coordinate for the window;

  • top {int} - y coordinate for the window;

  • width {int} - window width;

  • height {int} - window height.

2.16. setTitle(title)

This method sets the title for your window.

Parameter:

title {string} - window title.

2.17. setStatus(message, mode)

Apply this method to specify the status message that will appear in your window status bar.

Parameters:

message {string} - a message that will appear in the status bar.

mode {string} - additional attribute. If set to "temp", the previous value will be saved, if you specify "restore" as a value, the previous value will be retrieved.

2.18. setContent(text)

By using this method you may define a simple text as a window content.

Parameter:

text {string} - text to be used as a window content.

2.19. setDivContent(div)

This method sets the window content from the DIV element on the current page. The original element is cut from the DOM tree and put inside Window.

Parameter:

div {string | object} - a string or object that specifies the ID of the DIV element on the current page.

See the Advanced Usage section for instructions on how to preserve original DOM element with window content in its original place.

2.20. setContentUrl(url)

Call this function if you need to define the URL of the file from which the content should be retrieved.

url - link to the file (must have valid HTML content)

2.21. activate()

This method activates the window. The active window is indicated by having a different colored title bar. If it is already active, nothing will occur.

2.22. deactivate()

Deactivates the window. The window loses focus and changes its color scheme. If the window is already unactive, nothing will occur.

2.23. isVisible()

Returns true if the window is visible. False if hidden.

2.24. show()

Shows the window.

2.25. hide()

Hides the window.

2.26. showAtElement(el, opts)

Call this function to show the window at the specified anchor element.

el - ID of the anchor element

opts {string} - additional alignment options. See getState() for description. Optional.

2.27. showAt(x,y)

Invoke this function to show your window at the specified absolute position.

Parameters:

  • x {int} - x coordinate;

  • y {int} - y coordinate.

2.28. minimize()

Use the minimize() method to minimize your window.

2.29. maximize()

The method maximizes the window. Maximizing the window will expand the active window to fill the entire screen.

2.30. restore()

Call this function to restore your minimized or maximized window to its previous size.

2.31. close()

Closes the window, but doesn't delete the instance, just HTML portion of it.

3. Event Handlers

3.1. beforeWidthChange

An event handler that is fired when the window width is about to be changed by the corresponding window API.

As a parameter, the callback function receives the width that should to be set.

3.2. onWidthChange

An event handler that is fired when the window width is changed by the corresponding window API.

Receives the new width as a parameter.

3.3. beforeHeightChange

This event fires when the window height is about to be changed by the corresponding window API.

As a parameter, the function receives the height to be set.

3.4. onHeightChange

An event handler that is fired when the window height is changed by the corresponding window API.

The new height is passed as a parameter.

3.5. beforeResizeInit

An event handler that is fired right before resizing initialization has started. For example: when mouseDown event was fired and mouse pointer is on the window's border.

As a parameter, the callback function receives a DOM Event object.

3.6. onResizeInit

The event is fired right after resizing initialization has started.

As a parameter, the callback function receives a DOM Event object.

3.7. beforeResize

An event listener that fires right before the window size is changed by mouse action.

As a parameter, the callback function receives a DOM Event object.

3.8. onResize

An event handler that is fired right after window size is changed by the mouse action.

As a parameter, DOM Event object is passed to this callback function.

3.9. beforeResizeEnd

An event handler that is fired right before the resizing procedures are completed.

As a parameter, DOM Event object is passed to this callback function.

3.10. onResizeEnd

This event is fired right after the resizing procedures have been completed.

As a parameter, DOM Event object is passed to this callback function.

3.11. beforePositionChange

An event handler that is fired right before the window position is changed by corresponding API.

As parameters, the callback function receives new x and y coordinates to be set.

3.12. onPositionChange

This event fires right after the position is changed by corresponding window API.

As parameters, the callback function receives x and y coordinates.

3.13. beforeDragInit

Is fired right before dragging initialization has started. For example: when mouseDown event was fired and mouse pointer is on the window's title.

As a parameter, DOM Event object is passed to this callback function.

3.14. onDragInit

An event listener that is fired right after dragging initialization has started.

As a parameter, the callback function receives a DOM Event object.

3.15. beforeDragMove

An event listener that is fired right before the window position is changed by the mouse action.

As a parameter, the callback function receives a DOM Event object.

3.16. onDragMove

An event listener that is fired right after the window position is changed by the mouse action.

As a parameter, the callback function receives a DOM Event object.

3.17. beforeDragEnd

An event handler that is fired right before the dragging procedures being completed.

As a parameter, the callback function receives a DOM Event object.

3.18. onDragEnd

An event handler that is fired right after the dragging procedures being completed.

As a parameter, the callback function receives a DOM Event object.

3.19. onContentLoad

An event handler that is fired when the window content is loaded.

No parameters are passed to this callback.

3.20. onRaise

An event listener that is fired when the window becomes active.

No parameters are passed to this callback.

Chapter 5. Advanced Usage

1. Zapatec.Window.setup() and constructor

Including all possible properties in the constructor limits windows flexibility. For example, when you want to cancel some default action, you need to add specific property for that. That is why the constructor is only responsible for creating instance of an object, initializing it with properties and fetching theme and template.

Visual representation of the window and most of its methods are not available right after constructor has been called. For this, you have to call the create() method at first.

To make it easier for developer the setup() method has been introduced. It gives you ability to quickly initialize common configurations and extensions.

Several properties are to be used only with the setup() method, but not with constructor.

Constructor does not recognize the following configuration options: popup, triggerEvent, align, width, height, left, top, title, content, divContent, urlContent and initialState. They are initialized by the setup() method.

If you are not satisfied with the setup() method functionality, you can avoid using it by calling constructor itself, and then initializing appropriate methods manually.

For example, when the create() method is called window is being shortly shown on the screen regardless of the initialState property. To avoid this behavior you can call constructor in the HEAD section and create() and show() methods wherever you want.

2. Content Setting

2.1. Content Specified by a Simple String

This is the easiest way of specifying the content for your window. You may simply pass a string containing a text as an argument to the content option inside the Zapatec.Window.setup() method or use setContent() method. The string can contain HTML markup and JavaScript code that will be immediately executed.

2.2. Content Specified by DOM Object

Read carefully this section before you start implementing content this way.

Pass the ID of the DOM object as an argument to divContent option inside theZapatec.Window.setup() method or use setDivContent() method. When ID is passed, it will be searched in the document.

When found, it will be cut off the document and placed inside the window. JavaScript will not be executed as it was executed before this fragment was cut off, right after the page loaded.

If you want to use this element more than just a content for this window (for example, for other windows), then you should specify it using one of these:

divContent : myContentElement.cloneNode(true);

or

setDivContent(myContentElement.cloneNode(true));

Violation of the latter can cause unpredictable behavior of your script.

2.3. Content Set as URL

Note that setting content this way is the only way to do that in asynchronous mode.

Pass URL as an argument to the urlContent option inside Zapatec.Window.setup() method or use setUrlContent() method. Because content fetching is asynchronous you should start manipulation with it only onContentLoad. The execution of scripts is allowed. If iframeContent set to TRUE, then the content is a regular HTML with no restrictions. URL can point to the other domains (in this case its content can't be manipulated by your code). If iframeContent is set to FALSE, then you will not be able to pass URL from the other domain. We strongly recommend you to give only the content that can be valid BODY section of the page. If you insert other tags, then undesired behavior of your script can possibly occur.

2.4. iframe Warning

If iframeContent is FALSE when clicking on the link inside your window, then redirection will effect the whole page, not just window content. To avoid this, set the iframeContent option to TRUE.

3. Methods Scheduling

For the better functionality and to avoid crashes caused by asynchronous creation of the window instance, several specific methods require window to be in some certain state. Window's states are fully internal entities. The only information that is relative to user - "ready" state occurs after create() method was called and executed. The following methods can be called at any place in the program code, but will be executed under condition that "ready" state is reached: setState(), setPosition(), setWidth(), setHeight(), setTitle(), setStatus(), setContent(), setDivContent(), setContentUrl(), activate(), deactivate(), show(), hide(), minimize(), maximize(), restore(), close(), and other methods that use one of the listed above.

For example:

 var win = Zapatec.Window.setup({...});
 win.setTitle("some title");

the setTitle()method will be executed not right after the call, but when "win" will be created (in most cases onLoad event of the BODY). Methods scheduling allows to safely call these methods at any place inside your code. But you should never rely on immediate execution of them, since they will be executed only under some condition, or if it was never met - they will not be executed at all! For this reason these methods don't return anything. If window was closed (by user or by close()) and then you call some other method - it will not be put on schedule at all. But there are some other methods that are being executed right after they are called and return some value: for example those that start with "get" or "is".

4. Template

Inside the src folder, you can find the window-struct.js file. This file serves as an HTML template for your window. You can easily edit, add new features and extend your window structure using this file. Do not delete structures marked by class that starts with "area" as they are system structures, and you can cause further crash if you delete them. For now, there are no certain rules and limitations as to the use of HTML template. It is limited only by HTML rules and the presence of system structures, so you are free to experiment while creating your own window style and structure.

5. Listeners

There are two ways to add event listeners to certain window events. First of them is based on the use of eventListeners. For example:

Zapatec.Window.setup({
    ...
    eventListeners : {
        onShow : function() {},
        onClose : someFuncRef
    },
    ...
});

The other one is based on the use of addEventListener() method. For example:

 win.addEventListener("onClose", someFunc);

All other ways to add event listeners are not acceptable and exist only to ensure backward compatibility with previous versions.

6. Tips on Positioning and Sizing

If you have specified window coordinates through the constructor - then they will be treated as screen relative instead of page relative coordinates.

For setPosition(),setScreenPosition(), setWidth(), setHeight() and other methods that implicitly use methods listed above you can specify not only digital, but literal values as well. Available literal values for setPosition() and setScreenPosition() are:

"top" - related only to Y coordinate, positions top side of the element to the top of the parent element. For setScreenPosition() parent element is browser's screen.

"TOP" - related only to Y coordinate, positions bottom side of the element to the top of the parent element. For setScreenPosition() parent element is browser's screen.

"bottom" - related only to Y coordinate, positions bottom side of the element to the bottom of the parent element. For setScreenPosition() parent element is browser's screen.

"BOTTOM" - related only to Y coordinate, positions top side of the element to the bottom of the parent element. For setScreenPosition() parent element is browser's screen.

"left" - related only to X coordinate, positions left side of the element to the left side of the parent element. For setScreenPosition() parent element is browser's screen.

"LEFT" - related only to X coordinate, positions right side of the element to the left side of the parent element. For setScreenPosition() parent element is browser's screen.

"right" - related only to X coordinate, positions right side of the element to the right side of the parent element. For setScreenPosition() parent element is browser's screen.

"RIGHT" - related only to X coordinate, positions left side of the element to the right side of the parent element. For setScreenPosition() parent element is browser's screen.

"center" - positions center of the element to the center of the parent element. For setScreenPosition() parent element is browser's screen.

"CENTER" - positions top left edge of the element to the center of the parent element. For setScreenPosition() parent element is browser's screen.

any number with or without "px" - coordinate of the element inside parent element. For setScreenPosition() parent element is browser's screen.

Usually, window's parent is <BODY>.

For setWidth() and setHeight() use:

"auto" - size sets exactly to fit the content

(some_number)% - size in percents of the parent element

(some_number) or (some_number)px - size in pixels.

Chapter 6. Scenarios

1. Disable Windows Features

When the traditional window look does not properly suit your need, you can disable several features of your windows.

You may want to make your window get focus only when the title bar is clicked. For this purpose use raiseOnlyOnTitle boolean. By default, when set to FALSE, the users can make the window active by clicking anywhere on it. To achieve the opposite behavior provide the following customizations in your code: 

win1 = Zapatec.Window.setup({
  left:50, 
  top:130, 
  raiseOnlyOnTitle:true
});

It can be useful to disable the window status bar. To do this use the showStatus option. Set this option to FALSE and the status bar won't be displayed for the current window. For example, 

win2 = Zapatec.Window.setup({
  left:270, 
  top:310, 
  showStatus : false
});

You may also choose to disable specific buttons for your windows. If you need to disable the Maximize button for your window set the showMaxButton option to FALSE. Repeat the same with the showMinButton boolean if you want to have the Minimize button disabled. See the following code:

win3 = Zapatec.Window.setup({
  left:340, 
  top:380, 
  showMaxButton : false, 
  showMinButton : false
});

For more information on disabling specific Windows features, refer to the Disable Features of Windows demo.

2. Load Window Content from DIV

Zapatec Windows provide functionality that makes it possible to load content for your window from DIV that can contain, in addition to text, HTML markup including images, links, etc. To do this you should set an ID of the element that would be used as a Window content as an attribute for the divContent option. For example, 

win1 = Zapatec.Window.setup({
  left:280, 
  top:250, 
  width:450, 
  height: 220,
  divContent : 'myDiv'
});

Note that unlike browser windows, when the user clicks a link inside the window, the entire window page is redirected to that URL.

To be able to open links in the same Zapatec Window use iframeContent option that will specify iframe as a type of container for the window content.

For more information on loading content from DIV, see the Load Window Content from DIV demo.

3. Content Fetching via AJAX

You can also load the content for your window from URL using Zapatec AJAX Transport.

For example, we may use a URL referring to HTML where Menu and Windows are combined on the same page. The content can be loaded to the simple window or to the iframe-based window. The HTML in the simple window behaves the same as any other HTML on the page. Clicking on a link will reload the whole page and you can not include headers in the loaded page. HTML in the iframe window behaves as a separate page. Clicking on a link will load the page in the window. The page needs to be a full HTML page with headers.

4. Popup Window

Create a popup window and control its behavior using multiple options. You can set an element that will trigger your popup window. As a trigger element, you may choose a DIV, an HREF (link), a button or another HTML element using the popup option. Moreover, with the help of triggerEvent option you may specify the event that will activate the trigger.

You can also control where the popup window appears relative to the triggering element. This is easily done using the align option.

Since popups are standard windows, you can choose to exclude the standard minimize, maximize, close and resize elements. If there is no need to include a close button, you will need to provide a different way for the user to close the popup.

See the following code for the sample popup window configuration:

Zapatec.Window.setup({
  width:150, 
  height: 120,
  popup : "Div1",
  align : "CC",
  showMaxButton : false, 
  showMinButton : false,
  showCloseButton : true,
  canResize : false,
  showStatus : false,
  triggerEvent : "mouseover"
});

5. Alert/Confirm Windows

Using Zapatec AJAX Window you can easily create alert and confirm windows and customize them to suit your needs.

To customize Alert/Confirm window, use the constructor that should receive the object of configuration options for the window.

You can pass the following options to customize your alert/confirm window:

To set your window's size, use the height and width configuration options.

To control where the window appears on the screen, set window's left and top coordinates using the left and top options.

Provide the text that will appear in the window's title bar using the title option.

To set the contents to be displayed by your alert/confirm window, use the message option.

Use the lang option to define the language of your messages. By default, it is set to 'eng'.

Moreover, you can use the buttonTheme option to control how buttons appear in your alert/confirm window.

You may also want do exclude certain buttons from your window. To do this, set the following configurations options to FALSE: showMaxButton , showMinButton , canResize and showStatus .

For the confirm window, using the onConfirm option you should specify the function that will be called if you confirm the action.

Once customization is complete, run the createContent method to create the content of the alert/confirm window.

When it is done, call the alert function to show your alert window. This method is called from constructor and receives a message for the alert as a parameter.

For the confirm window, call the getResponse method that shows your window and adds onConfirm action. As parameters, it receives a custom confirm action and a message for the confirm window.
