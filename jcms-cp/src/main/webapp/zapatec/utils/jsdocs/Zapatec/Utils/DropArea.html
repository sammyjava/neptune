Zapatec.Utils.DropArea

Zapatec.Utils.DropArea(config)
    This is Zapatec.Utils.DropArea object definition. It represents most of the routines connected to dropping to some area and tracking global events connected to dragging of the object.
    Parameters:
      config - {object} - all parameters are passed as the properties of this object. Constructor recognizes the following properties of the config object \code prop. name | description ------------------------------------------------------------------------------------------------- container | {string or HTML element} Element to be made the droparea (required). dropName | {string} id of droparea listeners | recognizes the following listeners: | - onDrop - drop event listener; | - onDragInit - dragging start event listener; | - onDragOver - drag over the droparea event listener (is fired only first | time the element came over); | - onDragOut - drag out of the droparea event listener (is fired only first | time the element went out); | - onDragEnd - dragging end event listener; This is the new interface of this object, but for backward compability we leave old one, where all this parameters are passed as arguments in the following order: container, dropname, onDrop, onDragInit, onDragOver, onDragOut, onDragEnd. This way is depricated on the current moment.
