Chapter 1. Get the Zapatec DHTML Tooltips files

1. Download the Zip File

The DHTML Tooltips files are bundled in a zip file. Click the download link in the Tooltip section of Zapatec’s web site, and follow the instructions to download the file.

Save the file (zptooltip.zip) to your website’s root folder on your computer or server.

2. Unzip zptooltip.zip

  1. On your computer, go to the folder in which you saved zptooltip.zip.

  2. zptooltip.zip contains a folder called zapatec that holds all of the files you need to create your tooltips.

  3. Open zptooltip.zip to unzip it.

Chapter 2. Quick Startup

If you have not done so follow the instructions to download and unzip the Tooltip files.

1. Set Up Your Tooltip

In your web editor (Dreamweaver, UltraEdit, etc.), open the web page in which you want to place the tooltip. Insert the cursor before the ending </head> tag. Paste the following style path and script paths before the ending </head> tag:

<!-- Javascript Zapatec utilities file -->  
<script type="text/javascript" src="zapatec/utils/zapatec.js"></script>
<!-- Javascript file to support tooltips -->
<script type="text/javascript" src="zapatec/zptooltip/src/zptooltip.js"></script>

1.1. Path Warning

If the web page containing the tooltip is saved in the zptooltip folder, these lines will work without any changes. If, however, you save the web page in another location (such as directly under the website’s root folder), you need to adjust the path for these files.

1.2. Upgrade Warning

If you have been using a version of Zapatec tooltip that is older than version 2.0 and are now migrating to the current version, you need to adjust the header files.

The older utils.js, tooltips.js are no longer supported. To upgrade, replace the old lines:

<script src="utils/utils.js" type="text/javascript"></script>
<script src="utils/tooltips.js" type="text/javascript"></script>

with

<script type="text/javascript" src="zapatec/utils/zapatec.js"></script>
<script type="text/javascript" src="zapatec/zptooltip/src/zptooltip.js"></script>

The new zptooltip.js is a full widget and accepts only one argument in the constructor as any other Zapatec widget - the config param

In addition, the use of the LINK element for CSS inclusion in pre-2.0 versions is no longer supported. For example,if you used the winxp.css in the HEAD of the HTML page your pre-2.0 version as in:

<link href="themes/winxp.css" rel="stylesheet" />

you should delete this line. Instead, set the CSS using the theme option in the Zapatec.Tooltip constructor.

See the theme configuration option for details.

2. Attach the Zapatec Tooltip Object to your Presentation

In the HTML page, create a new Zapatec.Tooltip object to attach a tooltip to some element. Here is an example that attaches a Zapatec tooltip to a DIV whose ID is "container" and uses element with ID is "tooltip" for tooltip contents:

<!-- Tooltip target -->
<div id="container">
</div>

<!-- Tooltip contents -->
<div id="tooltip">
Visit Zapatec for all your ajax needs.
</div>

<script type="text/javascript">
  // Create a new tooltip
  new Zapatec.Tooltip({target: 'container', tooltip: 'tooltip'});
</script>

3. Copy the files to your web server

Using your editing or FTP program, copy or "put" your web page and the entire tree folder to the root of your website.

4. Test the tooltip application

Using your web browser, navigate to the web page that you created to include the Zapatec DHTML Tooltip. Congratulations! You have set up the most basic version of the Zapatec DHTML Tooltip! Now, on to the fun and exciting features you can change with this widget!

Chapter 3. Configuration Options

When you create a tooltip, you can pass a set of configuration options that affect the tooltip’s behavior.

A few configuration options are required:

  • an option that specifies the element we want to describe by the tooltip

  • an option that specifies the element that represents the actual tooltip

Other configurations are optional.

1. target (string or object)

The target is the element being described by the tooltip. The target is a required option.

The tooltip will be shown when the mouse hovers over the target element and will disappear as soon as it leaves element boundaries. If it is a string, the target value is the ID of the target element. If it is an object, the target is a HTMLElement DOM object.

Here are some examples:

From an element ID:

...
target: 'myTarget' // ID of target element

You can also define the tooltip target as a HTMLElement object using getElementById(): 

target: document.getElementById('myTarget') // Object of target

2. tooltip (string or object)

The tooltip specified the element or ID of the element that represents the actual tooltip being shown/hidden. The tooltip is a required option unless the content config option is used.

The tooltip element is being hidden as soon as tooltip widget is created. It is also set to use absolute positioning in order to allow appearing on arbitrary point on page. It is automatically being shown when the mouse hovers over the target element and disappears as soon as it leaves.

3. parent (string or object)

The parent specifies the element or ID of the element that will become a tooltip parent. The parent is optional. If omitted - document.body is used by default.

The tooltip widget will move the tooltip element to its new parent as soon as it is created. Moving tooltip elements into document.body works for most layouts and allows the tooltip to appear anywhere on page. If your layout has other requirements for tooltip boundaries you can provide your own parent element.

4. theme (string)

The theme option specifies the CSS used to present the tooltip.

The string argument is the name of the theme:

theme : 'winxp2'

or the absolute or relative path to the theme file

theme : 'themes/winxp.css'

The theme name is case-insensitive. Some themes are provided in the themes folder.

If the theme option is not specified, the default.css file in the themes folder is used.

To use your own CSS throughout the enterprise without specifying the theme option for each page, you can rename your CSS file "default.css" and move it to the themes folder.

5. themePath (string, default ’../themes’)

This is the path to the tooltip’s theme directory. If the path to the theme is passed as the argument to the theme option, the themePath is not needed.

The default works if you created the web page in zptooltip/demo or any other subdirectories under zptooltip. In other cases, set the themePath explicitly.

6. movable (boolean)

This option should be specified to make the tooltip move with the mouse pointer once it is shown.

If movable option is true the tooltip follows mouse movements once it is shown. It hides as soon as the mouse leaves the target element as usual.

If movable option is false the tooltip is shown under the mouse pointer and stays there until the mouse leaves the target element.

7. content (string)

This option specifies html content to be used as tooltip.

If specified this html content will be shown as our tooltip and tooltip config option can be omitted.

8. offsetX (integer)

Use this option to set the horizontal offset from the cursor position.

9. offsetY (integer)

This option defines the vertical offset from the cursor position.

Chapter 4. Scenarios

1. Simple Tooltip

The easiest way to generate a Zapatec Tooltip in JavaScript requires only one line of code.

We use the following code in the Tooltips Demo to create a Zapatec Tooltip.

The IDs for the DIV and are used later by the Javascript code to find the target and tooltip elements. 

<div id='link1' >
  Test of tooltip
</div>

<div id='tooltip1' >
  Tooltip for link1
</div>

This example uses the following configuration options:

  • target : ’link1’ - defines the target element

  • tooltip : ’tooltip1’ - defines the tooltip element

Here is an instance of the Zapatec Tooltip with these configuration options.

<script type="text/javascript">
        new Zapatec.Tooltip({target: 'link1', tooltip: 'tooltip1'});
</script>

2. Tooltips from HTML

Another easy way to generate tooltips is to define them in html.

We use the following code in the Grid Stock Demo to create a Zapatec Tooltips.

You need to create a DFN tag for every tooltip on the page. Place the tooltip contents inside the DFN tag and place the tag itself where you want a tooltip to appear on the page. When you call setupFromDfn it parses all the DFN tags and automatically creates a new Zapatec.Tooltip instance for each DFN tag. The tooltip content is deleted from the DFN tag and it becomes the tooltip target element.


<dfn title='Google'>
  More about <a href='http://www.google.com'>Google</a>
</dfn>

Once you've defined your DFN tags there's just one line you need to call to make them into full blown tooltips.

<script type="text/javascript">
// Initialize tooltips
Zapatec.Tooltip.setupFromDFN();
</script>
