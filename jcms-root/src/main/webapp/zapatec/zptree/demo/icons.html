Adding icons
  • This demo shows how to add icons to tree in defferent ways
  • Using defaultIcons config option you can add additional cell left to the node content with CSS class equal to config option value (first tree in demo)
  • After that using CSS you can style different states of node:
    • .tree-item-expanded .customIcon - this style will be applied to expanded branch
    • .tree-item-collapsed .customIcon - this style will be applied to collapsed branch
    • .tree-item .customIcon - this style will be applied to node without branch
  • Also you can add custom icons for each node.
  • If tree is constructed from JSON source - define fields expandedIcon, collapsedIcon, fetchingIcon, elementIcon
  • If tree is constructed from HTML source - just put some <IMG> tags inside node content
  • If tree prevCompatible config option is set to true (second tree - default behaviour) first <IMG> tag will be treated as collapsed icon and second as expanded icon.
  • If node has no subtree - first <IMG> tag will be treated as node icon
  • If prevCompativle is false (third tree) you must assign custom CSS classes to <IMG>:
    • expandedIcon - icon for expanded branch.
    • collapsedIcon - icon for collapsed branch.
    • fetchingIcon - icon for fetching branch which is fetching its content from server.
    • elementIcon - icon for node without subtree.
