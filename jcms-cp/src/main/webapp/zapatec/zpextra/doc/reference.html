Chapter 1. Get the Zapatec DHTML Extra files

1. Download the Zip File

DHTML Extra files are bundled in a zip file. Click on the download link in the Extra section of Zapatec web site, and follow the instructions to download the file.

Save the file (zpextra.zip) to your website root folder on your computer or server.

2. Unzip zpextra.zip

  1. On your computer, move to the folder where you saved zpextra.zip file.

  2. zpextra.zip file contains zapatec folder that holds all the files you will need to create your extra application.

  3. Open or double-click zpextra.zip to extract it.

Chapter 2. Quick Startup

If you have not done so, follow the instructions to download and extract the Extra files.

1. Set Up Your Extra

In your web editor (Dreamweaver, UltraEdit, etc.), open the web page where you want to place zpextra. Insert your cursor before the ending </head> tag.

Paste the following script paths before the ending </head> tag:

<!-- Javascript Zapatec utilities file -->  
<script type="text/javascript" src="zapatec/utils/zapatec.js"></script>
<!-- Optional Zapatec button file; needed for button extra -->  
<script type="text/javascript" src="zapatec/utils/button.js"></script>
<!-- Optional Zapatec pane file; needed for pane extra -->  
<script type="text/javascript" src="zapatec/utils/pane.js"></script>
<!-- Optional Zapatec popup file; needed for popup extra -->  
<script type="text/javascript" src="zapatec/utils/popup.js"></script>
<!-- Optional Zapatec dropdown file; needed for dropdown extra -->  
<script type="text/javascript" src="zapatec/utils/dropdown.js"></script>

1.1. Path Warning

If your web page that contains the extra application is saved in the zpextra folder, these lines work without any changes. If, however, you save your web page to another location (for example, directly under your website root folder), you need to adjust the path for these files.

2. Copy Files to your Web Server

Using your editing or FTP program, copy or move your web page and the entire tree folder to the root of your website.

3. Test the Extra Application

Using your web browser, navigate to the web page that you created to include the Zapatec DHTML Extra. You have set up the most basic version of the Zapatec DHTML Extra! Now, you can change this highly adaptable application with the help of multiple features provided.

4. Configuration Parameters

4.1. theme (string)

The theme option specifies the CSS used to present the Extra feature.

The string argument is a name of the theme:

theme : 'winxp'

It may also be the absolute or relative path to the theme file:

theme : 'themes/winxp.css'

The theme name is case-insensitive.

The themes folder provides a number of predefined themes that can be applied and further modified to suit your needs.

If the theme option is not specified, the default.css file in the themes folder is used.

To use your own CSS throughout the enterprise without specifying the theme option for each page, you can assign the "default.css" name to your CSS file and move it to the themes folder.

Chapter 3. The Extra Utilities

1. Progress Bar

To create a progress bar like that in the Zapatec.Progressbar demo, use Zapatec.ProgressBar class.

var progressBarSolid = new Zapatec.ProgressBar({
  parent : 'progressBarContainerSolid',
  minValue : 0,
  maxValue : 100,
  value: 38,
  width: 300,
  height: 30,
  theme: 'solid',
  showLabels: 'values',
  title: 'Items loaded: ',
  eventListeners:{cancel: function (progress) {
   alert('Cancelled at: ' + progress);
  }}
});

The progress bar options are described below.

1.1. parent (string)

The parent option sets a reference to the DOM element that is the progress bar container. The parent option is required.

1.2. minvalue (int, default 0)

The minvalue option is used to define the lowest possible value for the progress bar.

1.3. maxvalue (int, default 100)

Using the maxvalue option you may specify the highest possible value for the progress bar.

1.4. value (int)

Defines the current value for your progress bar. It may be set to display this value when the progress bar is initially displayed.

1.5. width (int)

By using this option, you can set the width of the progress bar in pixels.

1.6. height (int)

This option is used to customize the height of the progress bar in pixels.

1.7. theme (string, default 'default' )

This option sets the theme for the progress bar. You may choose any of the predefined themes in the zpextra\themes\progressbar folder.

You can create a new theme for your progress bar or customize the already existing one and save it in this folder.

1.8. showlabels (string, default 'none')

The showlabels option specifies whether the values of the running progress bar should be displayed as percentages or as raw values, or not to be displayed at all. The valid values for showlabels are as follows:

  • 'percents'

  • 'values'

  • 'none'

1.9. title (string)

By using the title option, you can set a label for your values. The label is displayed in front of the values specified when the showlabels option is not set to 'none'.

1.10. eventListeners (function)

The eventListeners option is a function that listens for an event that will be used to affect the display of the progress bar.

In the demo, this option is used to handle the Cancel event, which is generated when the user clicks the Cancel button.

2. Color Picker

To create a color picker like that in the Zapatec.ColorPicker demo, use Zapatec.ColorPicker class. 

var colorInputPicker = new Zapatec.ColorPicker({
  button : 'inputAnchor',
  inputField: 'color',
  color: '#CCCCFF',
  title: 'Custom title Color Picker',
  offset: 0
});

The color picker options are described below.

2.1. button (string)

The button option sets a reference to the DOM element that the user clicks to make a color picker pop up.

The button is optional.

2.2. inputField (string)

The inputField option specifies a reference to the DOM element where the hexadecimal value for the selected color is displayed.

If the inputField is an INPUT or TEXTAREA, the value it displays is set to the current color. In other cases, the current color is displayed in the element's innerHtml element.

The inputField is optional.

2.3. color (string, default '#000000')

Use this option to define the hexadecimal value of the color that is set by default for your color picker.

2.4. title (string, default 'Color Picker')

This option is used to set the title of your color picker.

2.5. offset (int, default 10)

This option sets the offset in pixels between the button element and the color picker window.

2.6. singleClick (boolean, default false)

This defines how to handle the color selection. If true, color is selected on a single click on the palette. If false, a single click makes the color active, and the color selection associates with a double click.

2.7. eventListeners (function)

The eventListeners option is a function that handles the user's selection from the color picker.

In the example below,

var colorEventPicker = new Zapatec.ColorPicker({
 button : 'bgAnchor',
 color: '#CCCCFF',
 eventListeners:{select: function (color) {
  document.getElementById('bgDiv').style.backgroundColor = color;
 })

the select function listens for the user to:

  • click the OK button in the color picker OR

  • click the color preview area in the color picker OR

  • double-click a color in the color picker's palette

In response to any of these methods of color selection, this handler changes the background of the container to the selected color.

3. Pane

Pane was created to allow widgets to manipulate content efficiently. It is a system widget for the internal use. Pane has the following configuration options:

3.1. parent (string or object)

Use this option to set the reference to the DOM element where the newly created Pane will be placed. By default - document. body.

3.2. containerType (string)

Required. Sets the container type for the Pane. The expected values are: div, iframe, current. These values denote the type of the pane to be created.

  • div - creates a new DIV element and adds it into parent;

  • iframe - creates a new IFRAME element and adds it into parent;

  • current - uses element in the parent option as a pane container.

3.3. sourceType (string)

See Zapatec.Widget documentation for this option.

3.4. source (string or object)

See Zapatec.Widget documentation for this option.

3.5. width (number)

This option enables you to set the width of the Pane in "px".

3.6. height (number)

The option defines the height of the Pane in "px".

3.7. autoContentWidth (boolean)

This option determines whether the width of the Pane will be defined automatically.

3.8. autoContentHeight (boolean)

This option enables you to determine if the height of the Pane will be set automatically.

3.9. id (string)

Optional. Using this option you may pass a string to search the Pane by the ID specified.

3.10. onlyInit (boolean)

This option disables calling of the create method of the Pane from init, so HTML structure will not be created after calling constructor.

3.11. showLoadingIndicator (boolean)

Determines whether a loading progress indicator is to be shown inside the pane during loading. Default is true.

3.12. overflow (string)

With the overflow option, you may set the pane content overflow style. Default is "auto".

3.13. iframeScrolling (string)

Using this option you can determine iframe scrolling attribute value.

3.14. Pane Methods

The most commonly used public methods applicable to the Pane are described below.

3.15. getContainer()

This method returns the reference to the Pane container (either DIV or IFRAME element).

3.16. getContentElement()

Returns the reference to the Pane element that can be used for working with content. If the containerType option evaluates to 'div', this method will return the DIV element itself. When the containerType option is set to 'iframe', the IFRAME.document.body is returned.

3.17. getIframeDocument()

If the containerType option possesses the 'iframe' value, this will return the reference to the iframe document object.

3.18. isReady()

The method returns true if the containerType option is set to 'iframe' and iframe element was created.

3.19. setContent(value)

Use this method to set the pane content to the specified value.

3.20. setWidth(newWidth)/setHeight(newHeight)

By using this method you may resize your pane to the specified dimensions.

3.21. getWidth()/getHeight()

This method returns the current pane dimensions.

4. Button

To create a button, use Zapatec.Button class. You can customize the button you create using the following configuration options:

4.1. image (string)

Use this option to set the image for a button. Default value is 'null'.

4.2. width (integer)

This option configures the width of the button. Default value is 'null'.

4.3. height (integer)

This option sets the height of the button. Default value: 'null'.

4.4. className (string)

Sets the class name to apply to a newly created button. Default value: 'null'.

4.5. style (string)

Sets the style to be applied to a newly created button. Default value: 'null'.

4.6. text (string)

This option specifies the text title to display on the button. If the image option has been specified, this text will be used as alternative title.

4.7. preloadImages (boolean)

If TRUE, preloads all images (image, overImage, outImage, downImage) for the button. Default value: TRUE.

4.8. overStyle (string)

Sets the style to apply to a button during mouseover event. Default value: null.

4.9. overClass (string)

Defines the class to add to a button during mouseover event. Default value: null.

4.10. overImage (string)

Configure this option to show the image during mouseover event. This will be done only if the image option was specified. Default value: null.

4.11. overAction (function)

This option specifies the function that will be called during mouseover event. Default value: null.

4.12. outAction (function)

This option specifies the function that will be called during mouseout event. Default value: null.

4.13. downStyle (string)

Defines the style to apply to a button during mousedown event. Default value: null.

4.14. downClass (string)

Specifies the class to add to a button during mousedown event. Default value: null.

4.15. downImage (string)

This option defines the image to show during mousedown event. This will be done only if the image option was specified. Default value: null.

4.16. downAction (function)

This option specifies the function that will be called during mousedown event. Default value: null.

4.17. downActionDelay (number)

Sets the delay before the next function will be called if the mouse remain pressed. Default: 500.

4.18. repeatEnabled (boolean)

This option determines if the function on mousedown event should be repeated after the specified interval. Default value: TRUE.

4.19. repeatAcceleration (number)

Sets acceleration of the downAction reiteration speed every other time. Default value: 0.8.

4.20. repeatStartSpeed (number)

Sets initial speed of the downAction reiterations. Default value: 55.

4.21. repeatMaxSpeed (number)

This option sets the maximum speed of downAction reiterations. Default value: 5.

4.22. clickAction (function)

This option specifies the function that will be called during onclick event.

4.23. theme (string)

This option defines the theme for the button.

4.24. themePath (string)

This option specifies the path to the folder where the theme files reside.

4.25. idPrefix (string)

Sets the prefix for HTML IDs that will be assigned to control elements.
