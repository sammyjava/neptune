Chapter 1. Get the Zapatec DHTML Editor Files

1. Download the Zip File

The DHTML Editor files are bundled in a zip file. Click the download link in the Editor section of Zapatec web site, and follow the instructions to download the file.

Save the zpeditor.zip file to the website root folder on your computer or server.

2. Extract zpeditor.zip

  1. On your computer, go to the folder where you saved the zpeditor.zip file.

  2. zpeditor.zip resides in the zapatec folder that contains all the files you will need to create your editor.

  3. Double click zpeditor.zip to extract it.

Chapter 2. Quick Startup

1. Set Up Your Editor

In your web editor (Dreamweaver, UltraEdit, etc.), open the web page where you want the editor placed. Paste the following script paths before the ending </head> tag:

<!-- Javascript Zapatec utilities file -->
<script type="text/javascript" src="zapatec/utils/zapatec.js"></script>

<!-- Javascript file to support Editor -->
<script type="text/javascript" src="zapatec/zpeditor/src/zpeditor.js"></script>

<!-- optional Javascript file to support tooltips -->
<script type="text/javascript" src="zapatec/zptooltip/src/zptooltip.js"></script>

<!-- optional Javascript file to support color picker -->
<script type="text/javascript" src="zapatec/utils/colorpicker.js"></script>

<!-- optional Javascript file to support insert special character -->
<script type="text/javascript" src="zapatec/zpwin/src/window.js"></script>

1.1. Path Warning

If your web page containing the editor is saved in the zpeditor folder, these lines will work without any changes. If, however, you save your web page in another location (for example, directly under your website root folder), you will need to specify the path for these files in the header script.

2. Create the Editor

The easiest way to create the editor is to insert the following code anywhere on your page: 

<script type='text/javascript'>
  new Zapatec.MinimalEditor({
    field: "target"  // This is the ID of your textarea field
  });
</script>

Considering the code above, you will also need to have a <textarea> element somewhere on your page in order to make it work. Pay attention that the Javascript code to call the Zapatec Editor must be provided after this text area is rendered. Moreover, the id of your textarea must be the same that you have set for the field config option during editor creation.

The following is an example of a textarea html you can insert in your page to make the code above work: 

<textarea id="target" rows="10" cols="80"><b>Some</b> <i>predefined</i> content</textarea>

3. Copy Files to Your Web Server

Using your editing or FTP program, copy your web page and the entire editor folder to the root of your website.

4. Test the Editor Application

Using your web browser, navigate to the web page that you created to include the Zapatec DHTML Editor. Congratulations! You have set up the most basic version of the Zapatec DHTML Editor. Now, you can change this highly adaptable application with the help of multiple features provided.

Chapter 3. Configuration Options

As mentioned earlier, you create an editor using the following syntax:

 new Zapatec.MinimalEditor({
  field: "target",
  theme: "default",
  toolbarElements: [...]
      });

To control the editor behavior, pass a set of configuration parameters when you create the editor.

This section describes the editor configuration options.

1. field (string or HTMLTextArea, required)

The field option specifies the TextArea element to be converted into an editor. The editor being created (including the toolbar) will fit inside the boundaries of this textarea.

The field option can be a string id of a textarea element on the page or an actual reference to the textarea DOM element.

This option is required.

2. toolbarElements (array)

The toolbarElements option controls what toolbar buttons will be available in this editor.

This option is an array of strings where each string represents a different toolbar button. The buttons are added to the toolbar in the order they appear in this array.

The following list represents strings recognized as elements of the toolbarElements array and their meanings:

  • maximize - A toggle button for maximizing/restoring the editor to full page boundaries.

  • fontName - A font name select field.

  • fontSize - A font size select field.

  • foreColor - A font color picker toolbar button.

  • backColor - A background color picker button.

    Note

    If you are using one of the color options above you will need to include the following script file in your page:

    <script type="text/javascript" src="zapatec/utils/colorpicker.js"></script>

  • insertLink - A button for inserting hyperlinks.

  • insertImage - A button for inserting images.

  • insertTable - A button for inserting tables.

  • insertHorizontalRule - A button for inserting horizontal rules.

  • insertSpecialChar - A button that invokes a character map window for selecting a special character to insert.

    Note

    If you use this option, you will need to include the following script file in your page:

    <script type="text/javascript" src="zapatec/zpwin/src/window.js"></script>

  • bold - A button for toggling bold style on the selected text.

  • italic - A button for toggling italic style on the selected text.

  • underline - A button for toggling underline style on the selected text.

  • justifyLeft - A button for making paragraphs left-justified.

  • justifyCenter - A button for making paragraphs centered.

  • justifyRight - A button for making paragraphs right-justified.

  • justifyFull - A button for making paragraphs full justified.

  • insertOrderedList - A button for inserting ordered numbered lists.

  • insertUnorderedList - A button for inserting unordered bulleted lists.

  • selectAll - A button for selecting all editor content.

  • copy - A copy button.

  • cut - A cut button.

  • paste - A paste button.

  • outdent - A button for decreasing indentation.

  • indent - A button for increasing indentation.

  • undo - An undo button.

  • redo - A redo button.

  • about - About this editor button.

  • browser - A button for previewing editor html content in a new browser window.

  • switcher - A toggle button to switch between HTML and WYSIWYG mode.

  • fetch - A button for loading editor html content from a remote location.

  • save - A button for sending editor html content to a remote location.

  • newPanel - This element will make the next button appear in a new toolbar panel.

  • newRow - This element will make the next button appear in a new row.

3. maximizedBorder (number, default: 6)

The maximizedBorder option specifies the border in pixels around the editor when it is in maximized state.

4. enableTooltips (boolean, default: false)

The enableTooltips option defines whether tooltips are to be created for all toolbar buttons. If you set this option to true, you will need to include the following script:

<script type="text/javascript" src="zapatec/zptooltip/src/zptooltip.js"></script>

5. dumpHtml (boolean, default: false)

The dumpHtml option specifies how the editor content is converted to html. The fastest way to do this is by using browser DOM innerHtml. It is also the most browser-compatible way. Pass true to this option to switch to an alternative custom method of dumping editor content root node into html.

If you use this option, you will have to manually include the following script in your file:

<script type="text/javascript" src="zapatec/utils/dump.js"></script>

6. generateXhtml (boolean, default: false)

The generateXhtml option determines whether the valid xhtml 1.0 strict output document is needed. Set this option to true to make the editor message the output html before returning it.

You will need to include the following script line manually if you are using this option:

<script type="text/javascript" src="zapatec/utils/html2xhtml.js"></script>

7. customUndo (boolean, default: true)

The customUndo option specifies how undo operates. If this option is true, a custom JavaScript undo stack will be maintained. Otherwise, execCommand will be used to call browser specific undo/redo behavior.

8. preserveImgSrc (boolean, default: false)

The preserveImgSrc option determines if any extra measures are to be taken to protect img src attributes from being resolved into absolute URLs under IE. If this option is set to true, a workaround that preserves original img src attributes becomes activated. This occurs only in IE.

9. preserveAnchorHref (boolean)

Under IE, relative anchor hrefs are automatically converted to absolute href URLs. Set this option to TRUE in order to pre-process the html every time the user switches from HTML to WYSIWYG mode and vice versa. Default is TRUE for IE, overwise, FALSE.

10. externalBrowserNoScript (boolean)

Setting this option to TRUE will remove all script tags from html being previewed when "View in external browser window" is pressed. Default is FALSE.

11. persistPath (string, default: '')

The persistPath option specifies the remote location to be used for storing to and fetching from html content. POST and GET http requests will be sent to the script at this path as result of save and load.

12. theme (string, default: 'default')

The theme option specifies the CSS used to present the editor.

The string argument is the name of the theme:

theme : "default"

or the absolute or relative path to the theme file:

theme : "themes/default.css"

The theme name is case-insensitive.

If the theme option is not specified, the default.css file in the themes folder is used. There are 2 editor themes provided in the themes folder: default and winxp.

To use your own CSS throughout the enterprise without specifying the theme option for each page by renaming default.css file and moving it to the themes folder.

13. fullPage (boolean, default: false)

This option inputs the whole document inside the editor including the root html tag.

13.1. Browser-Specific Customization

Reinstall keyboard and mouse listeners for the WYSIWYG document after switching to WYSIWYG in IE.

Remove contentEditable=true attribute from the <body> tags in getFullHtml when dumpHtml is set to FALSE in IE.

Add opening and closing <html> tags when the dumpHtml option is set to FALSE.

Always remove contents of the <head> tag in Safari (as we cannot change it and the default script needs to be removed every time).

Chapter 4. Scenarios

1. Minimal

We use the following code in the Minimal Editor demo to create a basic editor.

<textarea id="target" rows="10" cols="80"><b>Some</b> <i>predefined</i> content</textarea>
<script type="text/javascript">
 var objEditor = new Zapatec.MinimalEditor({
  field: "target",
  theme: "default",
  toolbarElements: [
   'fontSize',
   'newPanel',
   
   'bold',
   'italic',
   'underline',
   'newPanel',

   'justifyLeft',
   'justifyCenter',
   'justifyRight',
   'newPanel',
   
   'insertOrderedList',
   'insertUnorderedList'
  ]
 });
</script>

This creates an editor with only basic functionality available on its toolbar.

The following keyboard shortcuts are available in all editor configurations:

  • Ctrl-B - Toggles bold style on the current text selection

  • Ctrl-U - Toggles underline style on the current text selection

  • Ctrl-I - Toggles italic style on the current text selection

  • Ctrl-C - Copy

  • Ctrl-X - Cut

  • Ctrl-V - Paste

    Note

    To enable the Cut, Copy, and Paste features in Mozilla, please follow this link.

3. Save

We use the following code in the Editor Save/Load demo to illustrate how the editor content can be stored to and loaded from a remote server location.

<script type="text/javascript">
  new Zapatec.MinimalEditor({
    field: "target",
    theme: "default",
    toolbarElements: [
      ...
      'fetch',
      'save'
    ],
    persistPath: "processKey.php"
  });
</script>

This code illustrates how the editor html content can be stored to and fetched from a remote location. The persistPath config option specifies the URL to be used for that. To store the content, use a POST http request, and to fetch - use a regular GET request. A keyword parameter is passed to the server indicating the storage we are reading or writing to. Go to processKey.php for a reference implementation of a server-side store/load functionality.

4. Get Content as HTML String

You can easily obtain editor content as an HTML string by using the following statement:

<script type="text/javascript">
  var html = objEditor.getHTML();
</script>
